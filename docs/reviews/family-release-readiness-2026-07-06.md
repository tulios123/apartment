## 🔴 חסם — אשכול "מחרוזות מספריות מ-Supabase" (ממתין לאישורך)

**השורש:** עמודות `numeric(n,2)` חוזרות מ-Supabase כ**מחרוזות** בזמן-ריצה. כמה סיכומי-כסף השתמשו באופרטור `+` עליהן — מה ש**משרשר במקום לחבר**, ומייצר מספרים אבסורדיים שהמשפחה תראה:

| מסך | תסמין שהמשפחה רואה | קובץ |
|---|---|---|
| תזרים (שנה/טווח) | הכנסת שכר-דירה בתחזית משורשרת → סך-הכנסה NaN/ענק | `src/lib/projections.ts:109` |
| בית "צפי לסוף החודש" + תזרים הוצאות | פרמיית-ביטוח משורשרת (2 פוליסות) → הוצאה קבועה ענקית | `src/lib/projections.ts:151` |
| הון "מבנה המימון" | "בלנדד 0.0%" במשכנתא דו-מסלולית | `src/pages/wealth/FinancingStructure.tsx:35` |
| הון "חוב כולל" | חוב בטריליונים עם הלוואת-בלון (גם אחת) | `src/pages/wealth/FinancingStructure.tsx:42` |
| יתרת-משכנתא משולבת | יתרה מושחתת כשמסלולים מתחילים בחודשים שונים | `src/lib/mortgage.ts:95` |

הקוד **כבר יודע** לתקן את זה במקומות אחרים (`Number(t.amount)` עם הערה "numeric col → string; coerce before summing") — פשוט פוספס בשלושת ההוקים/המקורות האלה.
